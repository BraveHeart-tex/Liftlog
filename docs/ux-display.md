## Mobile UX

Fast · responsive · low friction · readable · mobile-native.

Optimize for thumb reachability, quick scanning, minimal typing, large touch targets, and predictable interactions. Compact layouts, clear hierarchy, obvious actions. No dense UI, hidden actions, tiny targets, or deep navigation chains.

---

## Data Display

Workout/progress UI prioritizes progression visibility and fast scanning.

```

60 × 8, 8, 7
+2 reps / +2.5 kg

```

Show previous performance, progress changes, and meaningful comparisons. Concise labels, readable numeric formatting, no verbose text or cluttered metrics.

---

## Interactions

Use `Pressable`. Minimum touch target: `p-3`. Large tappable areas, obvious affordances, simple predictable gestures. No nested touchables, tiny hit areas, or overloaded gestures.

---

## UI Code Output

Complete, working, mobile-safe. Semantic tokens, shared primitives, dark-mode support, realistic workout/steps sample data. No web-centric layouts, desktop assumptions, or placeholder-heavy mockups.
